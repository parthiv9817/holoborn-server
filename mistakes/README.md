# Mistakes Log

One file per day: `YYYY-MM-DD.md`. Written at 7pm alongside the diary.

## Rules

- One mistake per entry. Each entry has three parts:
  1. **What happened** — the actual mistake, in plain language
  2. **Why it happened** — root cause, not just the surface (assumption, missing check, fatigue, etc.)
  3. **Rule for next time** — the specific behavior change that prevents repeat
- If today had no mistakes, write a one-liner saying so. Don't fabricate mistakes to fill the file. But don't fake "no mistakes" days either — be honest.
- Mistakes here are about **process, decisions, judgment** — not typos or compile errors. "Forgot to push" is a mistake. "Got a TypeError" is a debug session.
- Re-read the previous 3-5 entries when starting a new dev day. If the same pattern shows up twice, that's a real signal — escalate it to CLAUDE.md or the diary's "Lessons written in blood" section.

## Why this exists

The dev journey produces lessons. Lessons that don't get captured get repeated. This folder is the second-pass review — the first pass is the diary, this is the part where we name the failure modes specifically and turn them into rules.
