## The "Plastic Figurine" Saga finally ends

Today closed Layer 5 of a 5-layer onion that's been blocking demo quality for ~6 weeks. Worth recording the full arc because it's the kind of debugging story that future-Claude (and future-Parthiv) should be able to reference:

1. **Layer 1 (early May, ~2 weeks of cross-AI debug)** — TRELLIS bakes shadows + highlights into BaseColor. No Unity shader can rescue it.
2. **Layer 2 (yesterday)** — Meshy v6 Retexture with `remove_lighting=true + enable_pbr=true` produces clean de-lit PBR. Backend shipped.
3. **Layer 3 (this morning's diagnosis)** — Unity's `BiasMaterialsForMatte` was a TRELLIS-era hack multiplying roughness 1.5× and forcing metallic to 0. Once Meshy PBR landed, this hack was undoing Meshy's careful work. **Closed today: disabled in both code default AND scene YAML serialized override.**
4. **Layer 4 (this morning's diagnosis)** — Quest scene has no IBL environment. Passthrough mode = no skybox visible, no reflection probe baked. PBR materials had nothing to reflect against. **Closed today: built `IBLBootstrap.cs` via `[RuntimeInitializeOnLoadMethod]` static class that procedurally generated a studio-gradient cubemap and assigned to `RenderSettings.customReflectionTexture`.**
5. **Layer 5 (this afternoon's discovery)** — Procedural cubemap had box-filtered mips, NOT GGX-prefiltered IBL. Rough fabric (cotton, denim) sampled too-sharp reflections from low mips → "wet plastic sheet" sheen. **Closed today: Parthiv downloaded `studio_small_09_4k.exr` from polyhaven, I fixed import .meta (Cube + Linear sRGB + Specular convolution + Android 1024 max), moved into Resources subfolder, wired IBLBootstrap to load via `Resources.Load<Cubemap>("studio_small_09_4k")` with procedural fallback.**

Each layer was invisible until the previous one was fixed. That's why this dragged for 6 weeks — you'd fix one layer, the next would surface, and it'd look like you'd made no progress.

## The afternoon iteration cycle

Spawned a research-agent on Meshy's rendering stack — best ROI move of the day. Agent identified:
- **Three.js** as the renderer (not Babylon, not model-viewer)
- **`RoomEnvironment`** as the IBL source — procedurally-generated soft studio room, baked through `PMREMGenerator`. **Critical insight: their cubemap has NO hotspots** — almost uniformly soft greys. That's why matte fabric reads matte (rough materials have no bright pole to reflect).
- **Khronos PBR Neutral** as tonemap (or ACES Filmic) — Unity URP "Neutral" is NOT the Khronos curve (name collision)
- Ranked recommendations: 5-min flatten cubemap, 15-min switch URP tonemap to ACES, 30-min replace procedural with real HDRI Reflection Probe, 60-min port Khronos PBR Neutral shader
- **85% ceiling on Quest** — last 15% is hardware-bound (Meshy has desktop GPU + 16-bit HDR FB + MSAA + no foveated rendering penalty; Quest can't match economically)

I iterated through 4 IBL rounds across the session:
- **Round 1** (TopColor 1.40, RefInt 0.85) — face lit naturally, fabric still plastic-sheen
- **Round 2** (TopColor 1.00, RefInt 0.30) — dimmer, slight improvement, Parthiv saw "made it worst" (genuinely was darker without much sheen reduction)
- **Round 3** (TopColor 0.55, RefInt 0.20) + ACES — never tested, would have been even darker
- **Round 4** (TopColor 0.85 + brighter mids/bottoms + RefInt 0.25 + AmbientIntensity 1.55 + ACES) — brighter+flat — applied but only briefly tested before HDRI swap
- **Final (HDRI):** Parthiv dropped `studio_small_09_4k.exr` into `Assets/HoloBorn/Lighting/`, I fixed meta + moved to Resources + wired IBLBootstrap auto-load. Real GGX-prefiltered IBL is now the production reflection source.
