## Decisions worth remembering

- **Apple ID drama is solvable in 60 sec via `developer.apple.com/download/all/`.** Any Apple/Epic/Meta SDK that can be installed via vendor account, FIRST check vendor direct-download. App Store / vendor account stores are ONE path, not the only path.
- **Skip-to-Hello-GLB beat Hello-Cube-first** because toolchain confidence had risen during editor work. The bisect insurance argument is real but only matters when toolchain has lots of unknowns left. After getting MetaXR loaded + project compiled + Project Settings persisted + glTFRuntime cold-compiled, "the toolchain works" had been validated in 80% of the ways it could fail. Skipping the cube-only package saved ~50-95 min for zero loss of debug-ability.
- **Path 2 (runtime load) over Path 1 (editor import).** glTFRuntime is runtime-only by design; the production HoloBorn pipeline loads GLBs from RunPod at runtime; using Interchange's Static Mesh import would have tested the wrong code path. The Blueprint logic we're writing now stays IDENTICAL when we switch from `glTFLoadAssetFromFilename` to `glTFLoadAssetFromUrl` later — write once, use twice.
- **Option III (button-press) over Option II (auto-spawn).** Pays the Enhanced Input wiring tax now while we have full attention, and the same pattern handles Phase 5's burst capture trigger. Reusable infrastructure, not throwaway spike code.
- **`Basic` template > `Empty` template** for any first-test scene. Empty has no Sky Atmosphere / Sky Light fill — drops you into pure black void. Basic ships with a complete PBR scene baseline. Default to Basic unless explicitly assembling lighting from zero.
- **Read every vendor script's source before running it.** SetupAndroid.command's hardcoded NDK 25 / build-tools 34.0.0 defaults would have wrecked the toolchain pin if run bare. Cost of `cat <script>`: 30 sec. Cost of running with stale defaults: 30+ min of "why won't the APK package."
- **Read every plugin's `.uplugin` before assuming it works on every platform.** Caught Meta XR's `["Win64", "Android"]` whitelist upfront → set expectations correctly + chose deploy-to-device workflow → avoided wasting hours on "why won't VR Preview work on Mac." 30 sec of file reading saved hours.
- **`brick by brick` is the user's working preference for high-stakes execution sessions.** Saved as a feedback memory. Don't queue parallel side-tasks during critical paths. Walk-away time is for breaks, not for adding more cognitive load. The user's gut catches dilution before mine does.

## Lessons written in blood

- **Confident wrongness with verifiable claims is the most expensive failure mode.** Every mistake in tonight's `mistakes/` file shares one property: 30-60 seconds of verification (open the URL, read the README, cat the script, look at the archive page) would have caught the error before it reached the user. The cost of being wrong is always 5-50x the cost of checking. Default to checking.
- **UE5 `.uproject` + plugin source in `Plugins/` = compile-on-open.** UE5's "Modules out of date, rebuild?" dialog is the standard plugin-integration UX. Not an error. Click Yes, walk away, plugin compiles. ~10-25 min on Intel Mac depending on plugin LOC.
- **UE 5.5 Interchange replaces the legacy glTF Importer.** Drag-and-drop GLB always goes through Interchange now, which produces Static Mesh + Material + Texture assets. There's no way to "drag-and-drop into glTFRuntime" because glTFRuntime is runtime-only by design. For runtime loading, the path is: keep the `.glb` as a raw file, package via "Additional Non-Asset Directories To Package", load at runtime via Blueprint.
- **Blueprint exec pins (white triangles) and data pins (colored circles) NEVER connect to each other.** Pure functions (Get X, Is Y, Find Z) have no exec pins. Executable functions (Spawn Actor, Add Mapping Context, Print) have exec pins on both sides. The graph divides into a sideways data flow + a forward exec flow, meeting at executable functions.
- **UE5 ships shortcut helpers for many common compose-chains.** Before specifying a verbose multi-node chain (GPC → GLP → GEILPS), search for shortcuts in the editor's search dialog. Often a single node does what looks like 3 nodes' worth of work. Untick "Context Sensitive" if the shortcut isn't appearing.
- **CR7 wallpaper choice today (Madrid + United + Sporting/Portugal era, three side-by-side) is data about the user's self-conception:** "I want to keep redefining myself across multiple chapters." Unity → UE5 → Vision Pro is the same arc. Most engineers pick one engine for life. This one sheds shells. Worth banking as a psychological data point — the wallpaper isn't decoration, it's the goal.
