## Bricks 1-4 of Stage 3 graceful arrival

Shipped in brick-by-brick order with parthiv sideloading between each. Each brick committed clean:

- **Brick 1** (sigil rise + retex vanish bottom-up tied to sigil Y): perfect first try
- **Brick 2** (free-fall + bounce + emission pulse + wobble): perfect first try. Real quadratic gravity (`v += -9.81 * dt`), 0.7s fall for 2.4m drop matched math exactly. ⚠️ emission pulse on sigil didn't fire — `_EmissionColor` property check failed at runtime; cosmetic, not blocking.
- **Brick 3** (rigged GLB respawn + foot alignment): first sideload showed feet floating 20cm above sigil. Root cause: Meshy's `SkinnedMeshRenderer.localBounds` extends below visible feet (phantom skeleton room). Fixed with `riggedBoundsToFeetOffset` Inspector field subtracted from foot alignment target. Visible feet now plant precisely.
- **Brick 3 polish** (size continuity): floating retex looked massive, grounded rigged looked midget. Discovered Meshy rigging shrinks mesh ~13%. Added runtime uniform scale match (retex height / visible rigged height = scale factor) — applied to rigged before alignment + effectivePhantomOffset also scaled. Sizes now match across stages.
- **Brick 4** (breath on Spine02): subtle chest oscillation, 5° peak after Parthiv reported the original 1.5° was invisible at MR viewing distance. Math: 1.5° at spine = ~1cm chest motion (invisible), 5° = ~3.5cm (evident).
