## Decisions worth remembering

- **The cube cloud IS the avatar being built in real-time, not random "processing" particles.** Each backend phase has a corresponding visual transition. The wait IS the experience. This is the philosophical anchor — "temporal drama, not dead latency."
- **300 cubes is conservative — likely too few for silhouette coverage.** Cube count + size are tunable knobs. Re-evaluate empirically when Phase G silhouette behavior fires (cluster on placeholder mesh, see how dense it reads).
- **Cyan with 90/10 white-accent ratio (not pure cyan dominant).** Vision Pro restraint. Avoids "default URP emissive" cyberpunk cliche.
- **Single CubeCloudManager.Update() for all 300 cubes — zero per-cube MonoBehaviour callbacks.** ChatGPT's perf catch. Would have tanked Quest 3 CPU otherwise.
- **Generic humanoid placeholder mesh (`Assets/HoloBorn/Models/silhouette_placeholder.fbx`, Mixamo Y Bot) is the silhouette target during P3-P4.** Real GLB doesn't exist during those phases. Resample from real GLB at P5 climax. Backend changes: zero.
- **Eye-glow killed; breath + gaze acquisition replaces it as the spawn climax beat.** Avatar takes a subtle first breath + head turns toward user as the avatar settles. More premium than 80s robot trope.
- **Audio is sparse, event-driven, with fade-to-near-silence between phases.** No 5-minute continuous loops (becomes tinnitus). Heartbeat pattern, not soundtrack.
- **Test discipline — the verification cost asymmetry rule applies as strongly as ever.** Claude claimed Phase A done before TestClient verification; Parthiv caught it. Wrote the test, all green. Same lesson as 2026-05-07 mistakes file: "when verification cost is <60s and being-wrong cost is >5min, ALWAYS verify before asserting."
- **Commit cadence rule tightened mid-session.** First refinement: phase-level commits, not micro-task commits. Second refinement: don't auto-commit AT ALL, even at phase boundaries — wait for explicit "commit this" approval. Memory rule updated. CLAUDE.md's "git commit after every working feature" overridden via instruction priority order.
- **Teaching unfamiliar concepts: lead with WHAT the code does (concrete), not HOW we verify it (abstract framework jargon).** First attempt at NUnit/Assert/TDD jargon failed; reset to explaining EaseOutCubic itself with ease-out vs linear motion analogy (car braking, ball settling). Pattern: meta-before-concrete is a teaching anti-pattern.
