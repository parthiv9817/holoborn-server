### Refactor — RunPod S3 download path

The actual GPU handler returns `{glb_volume_path, glb_size_bytes, elapsed_seconds}` and writes the GLB to a RunPod network volume exposed via S3-compatible API (NOT AWS S3). Refactored in 3 commits:

1. `feat(config): add RunPod S3 settings + GPT_IMAGE_MODEL toggle`
2. `refactor(runpod_client): fetch GLB via RunPod S3 API` — module-level `submit_job` / `poll_until_complete` / `download_glb` / `delete_remote_glb`. boto3 client cached via `@lru_cache`, `signature_version='s3v4'`, **explicit credentials** (no AWS env-var pickup), all sync boto3 calls wrapped in `asyncio.to_thread`. Magic-byte verification (`b'glTF'`) before returning. Background asyncio task in `/generate-multiview` so Quest gets a `task_id` immediately and `/status` reads from a dict instead of polling RunPod on every request.
3. `feat(main): validate S3 creds at startup` — lifespan calls `s3.head_bucket()`. Misconfig crashes at boot, not mid-demo.

### Bug found and fixed

`fastapi.UploadFile` is a *subclass* of `starlette.UploadFile`, but `request.form()` returns starlette instances. The `isinstance(value, fastapi.UploadFile)` check in `multipart_utils.collect_frames` always returned False, so `/generate-multiview` rejected every multipart request with "no frame_* fields found". Fixed by importing from `starlette.datastructures` instead. One-line commit.

## Decisions worth remembering

- **Public repo** for the Mac backend — green squares + parity with `holoborn-gpu` outweighs marginal "leak risk" since `.env` is verifiably not in history and `.gitignore` is right.
- **Background asyncio task pattern** for `/generate-multiview` — handler returns `task_id` in <100ms, all heavy work (portraitize, RunPod submit, poll, S3 download) runs in `asyncio.create_task`. Quest polls `/status` and reads a dict. No live RunPod polls per Quest poll.
- **Lifespan crashes loud** on bad S3 creds. Better to fail at `uvicorn` startup than mid-demo with the founder watching.
- **Don't touch `holoborn-gpu`** — it has a working CI/CD pipeline. If the Mac side seems to need a GPU-side fix, flag it and ask before doing anything.
- **Diary > memory** — context goes here in `diaries/YYYY-MM-DD.md`, not in `~/.claude/.../memory/`. Easier to review, easier to keep current, no context rot.
